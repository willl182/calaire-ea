# Matriz Maestra de Actividades - OSE 88

**Fuente**: `docs/cuentas/actividades_88.md`

**Objetivo**: usar las actividades contractuales como eje principal de organización, trazabilidad y reporte para mayo, junio y julio de 2026.

---

## Criterio de lectura

- Cada actividad es una unidad de control.
- Las evidencias se registran una sola vez y luego se cruzan con SGC, aplicativos, costeo o reuniones si aplica.
- La nomenclatura base para archivos es `aaaammdd-a##_nombre-archivo`.
- Si un soporte no se puede asociar a una actividad, no debe dominar la organización del mes.

## Orden recomendado para mayo

1. A1, A2, A3, A5 y A11 como frente principal.
2. A10 y A12 como control normativo y documental.
3. A4 para no dejar pendiente el soporte a proyectos.
4. A6, A7, A8 y A9 como soporte transversal.
5. A13 queda más cargada hacia junio y julio salvo que existan eventos tempranos.

---

## Vacíos que conviene vigilar

- No asumir que una evidencia solo pertenece al SGC: primero se ata a la actividad.
- No llenar mayo con reuniones si no producen cambio técnico o documental.
- No dejar los aplicativos como categoría separada: se registran como soporte de A1, A2, A3, A5, A9 o A10 según corresponda.
- No confundir el informe mensual interno con el formato oficial de cobro.
