# Technical Finding / Milestone

**Context**: OSE 88 operational structure in Logseq

## The Discovery

The cleanest operating model for this graph is a three-layer separation:

1. Contractual activities `A1` to `A13` as the stable frame.
2. Executable work as `TODO` blocks with explicit `deadline::`.
3. Time control as reusable weekly and daily templates.

This avoids mixing milestones, reminders, and evidence in the same layer and keeps the graph searchable and maintainable.
