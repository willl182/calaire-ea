# Technical Findings: P-PSEA-01 Protocol Gap Analysis and Improvement Plan

**Context**: Protocol Adjustment - P-PSEA-01 General EA Protocol
**Date**: 2026-02-08 17:53
**Session**: 260208_1706

## The Discovery

Análisis exhaustivo del Protocolo General de Ensayo de Aptitud (P-PSEA-01) frente a ISO/IEC 17043:2023 e ISO 13528:2022, revelando 5 brechas críticas que deben ser corregidas para garantizar conformidad normativa y validez técnica de los ensayos de aptitud para gases contaminantes criterio (SO2, CO, O3, NO, NO2).

**El protocolo actual menciona genéricamente "estadísticas robustas" y "consenso de participantes", pero NO especifica:**
- Qué algoritmo robusto usar (Algoritmo A, MADe, nIQR)
- Fórmulas y umbrales cuantitativos de homogeneidad/estabilidad
- Criterio de decisión cuándo usar z vs z'
- Cómo manejar grupos pequeños (n < 5 participantes)
- Cómo validar el valor asignado contra referencias independientes

**El aplicativo CALAIRE-APP (Manual v0.4.0) ya implementa estos algoritmos de ISO 13528, pero el protocolo no lo vincula formalmente.**

## Detailed Gap Analysis Results

### Brecha 1 (Blocking): Algoritmos robustos no especificados

**Estado actual:** El protocolo menciona "estadísticas robustas" pero no especifica cuál algoritmo usar.

**Requisito normativo:** ISO 13528 §6.5.2, Anexo C
- "El median, MADe, y nIQR se permiten como estimadores simples. El Algoritmo A transforma los datos mediante winsorización."

**Solución:** Crear sección "Métodos Estadísticos" con definición de:
- Algoritmo A (iterativo)
- MADe (Median Absolute Deviation)
- nIQR (Normalized IQR)

### Brecha 2 (Blocking): Criterios H/S cuantitativos ausentes

**Estado actual:** Solo se menciona que "las pruebas están descritas en los Procedimientos para cada contaminante".

**Requisito normativo:** ISO 13528 Anexo B.2.2
```
ss ≤ 0.3σpt
```
Justificación: La varianza entre muestras contribuye menos que 10% de la varianza para evaluación de desempeño.

**Solución:** Crear sección "Criterios de Homogeneidad y Estabilidad" con:
- Fórmula explícita de homogeneidad
- Fórmula de estabilidad: |x̄hom - x̄stab| ≤ 0.3σpt + 2u
- Acciones ante incumplimiento

### Brecha 3 (Required): Criterio de decisión z vs z' no formalizado

**Estado actual:** El protocolo menciona z' cuando "la incertidumbre es significativa" pero no define "significativa".

**Requisito normativo:** ISO 13528 §9.2.1, §9.5.1
```
Si u(xpt) ≤ 0.3σpt → incertidumbre despreciable (usar z)
Si u(xpt) > 0.3σpt → incertidumbre no despreciable (usar z')
```

**Solución:** Añadir en Definiciones:
- z'-score con fórmula: z' = (xi - xpt)/√(σpt² + u²(xpt))
- Umbral de incertidumbre: 0.3σpt

### Brecha 4 (Blocking): Tratamiento de grupos pequeños (n < 5)

**Estado actual:** No menciona procedimientos específicos para bajo número de participantes.

**Requisito normativo:** ISO 17043 §7.2.2.3b, ISO 13528 §5.4
"El diseño estadístico deberá considerar el número mínimo de participantes necesarios y establecer enfoques alternativos."

**Solución:** Crear sección "Contingencias Estadísticas" con:
- Procedimiento cuando n < 5
- Usar valor de referencia (MRC o fotómetro nivel 2)
- Usar σpt predeterminado (no dispersión observada)
- Justificación y documentación

### Brecha 5 (Suggestions): Compatibilidad metrológica no mencionada

**Estado actual:** El protocolo menciona trazabilidad y MCR, pero no describe validación del valor asignado.

**Requisito normativo:** ISO 13528 §7.8
"Cuando exista valor de referencia independiente, el valor asignado por consenso debe compararse. Si |xdiff| > 2udiff, investigar."

**Solución:** Crear sección "Validación del Valor Asignado" con:
- Fórmula de compatibilidad metrológica
- Criterio de aceptación
- Causas posibles de discrepancia
- Vinculación con función de CALAIRE-APP

## Criterios Normativos Extraídos

### Homogeneidad y Estabilidad

| Criterio | Fórmula | Referencia |
|----------|---------|------------|
| Homogeneidad | ss ≤ 0.3σpt | ISO 13528 B.2.2 |
| Incertidumbre VA despreciable | u(xpt) ≤ 0.3σpt | ISO 13528 §9.2.1 |
| Estabilidad | \|x̄hom - x̄stab\| ≤ 0.3σpt + 2u | ISO 13528 B |

### Puntuaciones de desempeño

| Score | Fórmula | Interpretación | Referencia |
|-------|---------|---------------|------------|
| z-score | z = (xi - xpt)/σpt | \|z\| < 2.0 acep., 2.0 ≤ \|z\| < 3.0 advert., \|z\| ≥ 3.0 acción | ISO 13528 9.4.1 |
| z'-score | z' = (xi - xpt)/√(σpt² + u²(xpt)) | Mismos umbrales | ISO 13528 9.5.1 |
| ζ-score | ζ = (xi - xpt)/√(u²(xi) + u²(xpt)) | Evalúa consistencia con incertidumbre declarada | ISO 13528 9.6.1 |
| En-score | En = (xi - xpt)/√(U²(xi) + U²(xpt)) | -1.0 < En < 1.0 acep., \|En\| ≥ 1.0 acción | ISO 13528 9.7.1 |

### Compatibilidad metrológica

| Situación | Criterio | Acción |
|------------|---------|--------|
| xconsenso vs xref \|xref - xconsenso\| > 2√(u²(xref) + u²(xcon)) | Investigar causas (sesgo en referencia, sesgo común en participantes, etc.) | ISO 13528 §7.8.2 |
