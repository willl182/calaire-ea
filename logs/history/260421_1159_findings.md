# Technical Finding / Milestone

**Context**: Prototipo de formulario codificado para captura de resultados de participantes en CALAIRE-EA

## The Discovery

Para este flujo de trabajo, Google Forms no es apropiado porque obliga a un diligenciamiento lineal y repetitivo cuando el participante debe reportar múltiples contaminantes y niveles. La solución correcta es una interfaz tabular reactiva controlada por una configuración previa de ronda.

El prototipo implementado resuelve esa necesidad con una sola aplicación HTML/JS que contiene dos funciones separadas:

1. Un dashboard del organizador para definir la ronda.
2. Un portal del participante que se adapta automáticamente a la ronda elegida.

La ronda se modela como un objeto con:

- `code`
- `name`
- `openDate`
- `closeDate`
- `status`
- `description`
- `gases`
- `levelCount`
- `unit`
- `secondAverage`

Ese modelo permite que la interfaz del participante renderice dinámicamente:

- pestañas por contaminante
- filas por nivel
- columnas opcionales de `promedio_2` e `incertidumbre_2`

Sin backend, se dejó una capa transitoria de persistencia con `localStorage` y exportación/importación JSON para las rondas, además de exportación CSV para respuestas del participante.
