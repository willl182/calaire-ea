## Learnings to Preserve

1. **Revisión de fase sistemática**: La ejecución de `revisor-fase` después de cada fase es esencial para mantener calidad y consistencia. El proceso debe seguir el ciclo: implementar → revisar → corregir → commit/push.

2. **Convenciones explícitas**: La estandarización de propiedades (status en inglés sin #) es crítica para queries efectivas en Logseq. La plantilla en `pages/Laboratorios.md` debe ser mantenida como fuente de verdad.

3. **Indentación crítica**: Logseq requiere estrictamente tabs para jerarquía de bloques. El uso de espacios rompe la estructura y oculta propiedades de las consultas.

4. **Workflow de fases**: La estructura del plan con Log de Ejecución y Resultados de Revisión permite trazabilidad completa y aprendizaje continuo.

5. **Git workflow por fase**: Commit y push después de cada fase permite recuperación incremental y mantiene historial granular.
