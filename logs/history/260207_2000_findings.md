## Critical Technical Conventions Established

### Logseq Technical Standards
1. **Indentación**: Tabs (NO espacios) - CRÍTICO para estructura del grafo
   - Comando de corrección: `sed -i 's/^    /\t/g' filename.md`
   - Validación: `cat -A filename.md` para visualizar tabs vs espacios

2. **Sintaxis de Enlaces**:
   - Correcto: `[[Page]]` o `[[Page|Display Text]]` (alias)
   - Incorrecto: `[[[Page]]]` (no es sintaxis estándar)

3. **Status**: Inglés sin # (`confirmed`, `cancelled`, `planificada`, `pending`, etc.)

4. **Tags**: Inglés con [[]] (`[[Team]]`, `[[Laboratorio]]`)

5. **Cancelled Items**: Deben tener "(CANCELLED)" en título para visibilidad en navegación/backlinks

### Mermaid Diagram Standards
1. **Gantt IDs Únicos**: Cada tarea debe tener ID único por ronda
   - Patrón: `r[1-8]_[rec|ac|ini|cont|fin|dev]`
   - R1: r1, r1_rec, r1_ac, r1_ini, r1_cont, r1_fin, r1_dev
   - R2: r2, r2_rec, r2_ac, etc.

2. **Milestones**: Formato `task : milestone, id, date, 0d`
   - Ejemplo: `Inicio Prueba Piloto: milestone, m1, 2026-03-16, 0d`

3. **Fechas Consistentes**: Gantt y Timeline deben usar las mismas fechas

4. **Visual Cancellation**: Marcar rondas canceladas con "(CANCELLED)" en sección titles

## Learnings to Preserve

1. **Avoid scheduling near environmental contingency periods**: February-March is critical for SIATA and environmental authorities like Corantioquia.

2. **Logseq requires tabs for indentation**: Spaces break structure and affect queries. Always validate with `cat -A`.

3. **Mermaid Gantt IDs must be unique**: Reusing IDs causes rendering issues or collisions. Use consistent naming pattern.

4. **Link consistency**: Use same page names across files to avoid creating duplicate pages.

5. **Milestones in Mermaid require specific format**: Always use `task : milestone, id, date, 0d`.

6. **Cancelled items need visual markers**: "(CANCELLED)" in titles improves navigation and backlink visibility.
