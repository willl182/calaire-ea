# Technical Finding / Milestone: Corrección Final de Estructura PMI

**Context**: Ajuste final de ubicaciones de documentos PMI y actualización de AGENTS.md con estructura definitiva

## The Discovery

El usuario aclaró que los artefactos PMI deben ir en `docs/pm/` (carpeta nueva), separando así:
1. **Artefactos PMI** → `docs/pm/` (NUEVA carpeta)
2. **Guías PMI** → `docs/instrucciones/` (carpeta existente con guías)
3. **Archivos operativos CSV** → `docs/auxiliares/` (carpeta existente)

Esto permite una separación clara entre:
- Guías de referencia (instrucciones para desarrollo)
- Artefactos de gestión formal (documentos PMI)
- Archivos operativos y datos (CSV, calendarios)

## Impacto en Workflow de Desarrollo

1. **Separación de responsabilidades:**
   - Guías de referencia → `docs/instrucciones/`
   - Artefactos formales → `docs/pm/`
   - Archivos operativos → `docs/auxiliares/`

2. **Mantenimiento de compatibilidad:**
   - Cronograma consolida archivos existentes
   - Guías existentes siguen siendo fuente de verdad
   - MOC central en `pages/` conecta todo

3. **Escalabilidad:**
   - Estructura permite agregar nuevos artefactos sin afectar guías
   - CSV operativos independientes de artefactos formales
   - Scripts de automatización en directorio separado

## Reglas para Futuros Agentes

1. **Al crear artefactos PMI**: Siempre crear en `docs/pm/`
2. **Al consultar guías**: Usar `docs/instrucciones/guide_pm_charter.md` (Charter) o `guide_pm_wbs.md` (EDT)
3. **Al consolidar cronograma**: Usar `docs/auxiliares/gantt.md` + `timeline.md` existentes
4. **Al crear CSVs**: Siempre crear en `docs/auxiliares/`
5. **Al actualizar MOC**: Enlazar artefactos en `docs/pm/` desde `[[Gestión del Proyecto]]`
6. **Scripts de automatización**: Siempre crear en `scripts/pm/`
