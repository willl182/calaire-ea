# Technical Finding: Configuración Inicial del Sistema de Gestión CALAIRE-EA

**Context**: Logseq Knowledge Graph + Git Repository Setup

## The Discovery

Se implementó exitosamente un sistema híbrido que combina:

1. **Logseq como PKM**: Grafo de conocimiento para gestión de proyecto científico/metrológico.
2. **Git como VCS**: Control de versiones para colaboración y auditoría.

### Arquitectura de MOCs Implementada

```
CALAIRE-EA (Hub)
├── Prueba Piloto (Operación)
├── CALAIRE-APP (Software)
├── QMS (Calidad)
├── Laboratorios (Datos)
└── Equipo (Personas)
```

### Queries Avanzados Configurados

Los queries en `config.edn` usan Datalog para filtrar bloques por referencia de página:

```clojure
;; Busca TODOs que referencian [[CALAIRE-EA]]
[:find (pull ?b [*])
 :where
 [?b :block/marker ?m]
 [(contains? #{"TODO" "DOING"} ?m)]
 [?b :block/refs ?p]
 [?p :block/name "calaire-ea"]]
```

### Convención de Propiedades

| Propiedad | Uso | Ejemplo |
|-----------|-----|---------|
| `priority::` | Prioridad de tarea | `high`, `medium`, `low` |
| `deadline::` | Fecha límite | `2026-02-20` |
| `status::` | Estado de documento | `#borrador`, `#revisión` |
| `project::` | Vinculación a proyecto | `[[CALAIRE-EA]]` |
