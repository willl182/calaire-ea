# Technical Finding / Milestone

**Context**: CALAIRE-EA Knowledge Graph — Sistema de clasificación de contenido journal

## The Discovery

Se diseñó e implementó un sistema de dos etiquetas (`#internal` e `#insight`) para clasificar el contenido de los journals y distinguir entre:

1. **Trabajo interno del dev/PL** → `#internal` (repo cleanup, scripts, skills, migraciones, refactoring)
2. **Descubrimientos y correcciones de entendimiento** → `#insight` (conocimiento adquirido, no entregable)
3. **Todo lo demás** → sin etiqueta (reportable por defecto en reuniones de avance)

### Artefactos creados/modificados

| Archivo | Cambio |
|---------|--------|
| `AGENTS.md` | Nueva sección "Content Classification Tags" con tabla, 5 reglas de aplicación, y posición de etiquetas |
| `AGENTS.md` | Skills table actualizada con `tagger-journal` y `evaluador-tagging` |
| `AGENTS.md` | Phase Workflow ampliado con documentación de ambos subagentes, ejemplos de invocación, y cadena típica |
| `ref/subagentes.md` | Definición formal de `tagger-journal` (criterios jerárquicos, output, restricciones) y `evaluador-tagging` (protocolo, niveles de severidad, veredicto) |

### Ejecución de la cadena de subagentes

- **Tagger-journal**: Revisó 25 journals (abril–mayo 2026), identificó ~80+ bloques, aplicó 10 etiquetas.
- **Evaluador-tagging**: Auditó el resultado. Veredicto: **Aprobar**. 0 errores Blocking, 0 Required, 5 sugerencias de mejora estructural para futuros journals.
