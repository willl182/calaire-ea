# Technical Finding / Milestone: Estructura Correcta de Carpetas para PMI

**Context**: Revisión de ubicación de documentos PMI después de reorganización del usuario

## The Discovery

El usuario ha reorganizado la estructura del repositorio. Los artefactos PMI NO deben crearse en una nueva carpeta `docs/pm/` como se asumió inicialmente. Las ubicaciones correctas son:

1. **`docs/instrucciones/`**: Guías y documentación PMI (instrucciones para desarrollo de actividades)
2. **`docs/auxiliares/`**: Archivos operativos, CSV, calendarios y archivos útiles al proyecto

## Reglas de Navegación

Para futuros agentes que trabajen en este proyecto:

1. **Artefactos PMI**: Siempre crear en `docs/instrucciones/`
2. **Archivos CSV**: Siempre crear en `docs/auxiliares/`
3. **Guías PMI**: Referenciar `guide_pm_charter.md` (Charter) y `guide_pm_wbs.md` (EDT) antes de crear artefactos
4. **Cronograma**: NO crear nuevo `05_Cronograma.md` desde cero; CONSOLIDAR `gantt.md` y `timeline.md` existentes
5. **CSV Bridge**: Script `sync_appsheet.sh` debe apuntar a `docs/auxiliares/`
