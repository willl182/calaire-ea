## The Discovery

### Esqueletos de documentos urgentes creados (Oleada 1 + pre-ronda)

Se diseñaron y escribieron 6 esqueletos detallados que convierten los placeholders de 28–39 líneas en estructuras de documento completas (100–200 líneas) con instrucciones de redacción embebidas.

#### Patrón de estructura aplicado a instructivos (I-PSEA)

Todos los instructivos siguen esta estructura estándar:

```
1. Objeto           → instrucción de redacción en 1-2 oraciones
2. Alcance          → delimitar analitos, rondas, condiciones
3. Referencias      → tabla norma/cláusula
4. Definiciones     → tabla con términos técnicos pre-llenados
5. Responsabilidades → tabla por rol
6. Procedimiento    → pasos numerados con > Instrucción: para cada sección
7. Criterios de aceptación → tabla criterio/condición/resultado
8. Registros        → tabla formato/descripción/cuándo
9. (opcional) Diagrama Mermaid
   Control de versiones
```

Las instrucciones embebidas usan el formato `> **Instrucción:**` dentro de cada sección, lo que permite al redactor saber exactamente qué escribir sin necesidad de un documento externo de guía.

#### Hallazgos específicos por documento

**I-PSEA-07 (Diseño Estadístico):** Se identificó que la prueba piloto con n=1–2 requiere una sección especial (§9) que documenta explícitamente las decisiones estadísticas para n bajo: siempre x_pt de referencia trazable (no consenso), siempre σ_pt prescrito, score z' o ζ. Esto evita confusión durante el análisis post-ronda.

**I-PSEA-08 (Valor Asignado):** El modelo de incertidumbre combinada u²(x_pt) = u²_char + u²_hom + u²_stab quedó explícito con las derivadas parciales de la fórmula de dilución. La sección §6.5 de trazabilidad CRM incluye tabla para documentar proveedor, certificado, organismo de acreditación y fecha de vencimiento.

**I-PSEA-13 (Validación de Software):** Se definieron 5 datasets de referencia obligatorios, incluyendo los ejemplos del Anexo C de ISO 13528:2022 (DS-01, DS-02) como dataset primario de aceptación. La tolerancia numérica de ≤ 1×10⁻⁹ queda como criterio formal. Tabla de tipos de cambio que obligan a revalidación total vs. parcial.

**F-PSEA-05A (Hoja de Registro del Participante):** Archivo nuevo creado desde cero. Basado en benchmarking ERLAP/Barbiere del final_plan.md. Incluye sección interna (§6) para uso exclusivo del coordinador (no enviar al participante). Listo para enviar antes del 13-abr.
