# Technical Finding / Milestone

**Context**: Definición de arquitectura objetivo para autenticación, perfiles y gestión de rondas del portal de participantes de CALAIRE-EA

## The Discovery

La adopción de `WorkOS` es apropiada para resolver identidad, invitaciones, sesiones y pertenencia organizacional de los laboratorios participantes, pero no debe absorber la lógica operativa del servicio de ensayo de aptitud. La separación correcta consiste en:

- `WorkOS` para autenticación y organizaciones
- `PostgreSQL` para rondas, perfiles, asignaciones y envíos
- `Next.js` para unificar el portal `admin` y el portal `participante`

Esta decisión permite evolucionar el prototipo actual sin perder el modelo tabular ya validado conceptualmente, y evita que la asignación de rondas quede acoplada a una herramienta externa de identidad.
