### 1. Laboratory Status in Prueba Piloto.md

**Initial Consideration**: Should laboratory confirmations in `[[Prueba Piloto]]` be plain text or wiki links?

**Decision Made**: Use wiki links `[[Laboratorio Name]]` to individual lab pages.

**Justification**:
- Provides navigation to detailed lab information
- Maintains consistency with Logseq graph philosophy
- Allows tracking of confirmations from both perspectives
- Prueba Piloto shows "who is confirmed by round", lab pages show "which rounds this lab participates in"

**Implementation**: Updated `[[Prueba Piloto]]` line 34-36:
```
- **Confirmaciones de Laboratorios**
    - [[Universidad Pontificia Bolivariana]] :: Carta enviada
    - [[Universidad de Medellín]] :: Confirmada para Rondas 3 y 4
```

### 2. Participation Rounds Structure

**Initial Consideration**: How to structure round participation in lab pages?

**Option A**: Single block listing all rounds with status
**Option B**: Separate blocks per round with details

**Decision Made**: Option A - Simple list for current implementation.

**Justification**:
- Simpler for initial population
- Can expand to Option B later if detailed tracking is needed
- Matches current information availability

**Implementation**:
```
- ## Participación en Rondas EA
    - [[Ronda 1]]: #pendiente
    - [[Ronda 2]]: #pendiente
    - [[Ronda 3]]: #confirmado (18-23 marzo 2026)
    - [[Ronda 4]]: #confirmado (25-30 marzo 2026)
    - [[Ronda 5]]: #pendiente
```

## Recommendations

1. **Future Lab Pages**: When adding new laboratories, always follow the standard structure documented in ref/setup.md section 2.5.

2. **Equipment Inventory**: Populating the equipment section should be done in coordination with lab contacts to ensure accuracy (analyzer models, calibration dates).

3. **Documentation Links**: Each lab page should eventually include links to:
   - Calibration certificates
   - Participation agreements
   - Technical protocols specific to that lab

4. **Status Updates**: When a lab changes status (candidate → contacted → confirmed), update both:
   - The lab page (`status::` property)
   - The MOC `[[Laboratorios]]` (move to appropriate section)
