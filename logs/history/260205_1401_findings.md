# Technical Finding / Milestone

**Context**: Corrección de convenciones en journal diario del proyecto CALAIRE-EA (Logseq knowledge graph)

## The Discovery

Al crear el journal diario `journals/2026_02_05.md` para registrar noticias del proyecto, se identificaron desviaciones de las convenciones establecidas para el grafo de Logseq que podrían afectar la visualización y consultas:

1. **Indentación incorrecta:** Las propiedades de bloques (date::, category::, status::) estaban indentadas con espacios en lugar de tabs. Logseq requiere indentación con tabs para mantener la jerarquía correcta de bloques.

2. **Encabezado sin prefijo de bloque:** El título `# Registro del Día` no tenía el prefijo `- ` al inicio, por lo cual no constituía un bloque válido de Logseq.

3. **Inconsistencia en idioma de propiedades:** Los valores de `status::` usaban tags en español con prefijo `#` (ej: `status:: #confirmado`) en lugar de valores en inglés sin tag (ej: `status:: confirmed`), fragmentando las queries de tareas y decisiones.

4. **Proyecto vinculado incorrecto:** El nuevo TODO agregado en `pages/CALAIRE-APP.md` solo tenía `project:: [[CALAIRE-APP]]` sin incluir `project:: [[CALAIRE-EA]]`, lo cual podría excluirlo de las queries globales del proyecto principal.

## Solution Applied

1. Normalizar indentación usando tabs únicamente para sub-bloques y propiedades
2. Convertir encabezado a bloque válido: `- # Registro del Día`
3. Estandarizar `status::` a valores en inglés sin prefijo `#`:
   - `#confirmado` → `confirmed`
   - `#pendiente` → `pending`
   - `#en_estudio` → `in_study`
   - `#en_revision` → `in_review`
   - `#en_proceso` → `in_progress`
4. Agregar `project:: [[CALAIRE-EA]]` al nuevo TODO para incluirlo en queries globales

## Key Files Affected

- `journals/2026_02_05.md` - Corrección completa de estructura y convenciones
- `pages/CALAIRE-APP.md` - Agregado `project:: [[CALAIRE-EA]]` al TODO de revisión de César Yate
- `logs/plans/260205_1356_plan_coordinacion-labs-app-roles.md` - Documentación de hallazgos del revisor-fase

## Learnings to Preserve

- **Regla de indentación Logseq:** Siempre usar tabs para sub-bloques y propiedades, nunca espacios
- **Convención de propiedades status:** Usar valores en inglés sin prefijo `#` (confirmed, pending, in_progress, etc.)
- **Convención de encabezados en journals:** El título principal debe ser un bloque con prefijo `- `
- **Proyectos múltiples:** Los TODOs deben vincularse tanto al subproyecto específico como al proyecto principal `[[CALAIRE-EA]]` para visibilidad en queries globales
- **Validación con revisor-fase:** Ejecutar subagente `revisor-fase` después de cada fase para detectar inconsistencias técnicas antes de continuar

## Checklist de Verificación para Futuros Journals

- [ ] Encabezado principal tiene prefijo `- ` (es un bloque válido)
- [ ] Todas las propiedades (date::, category::, status::, etc.) usan tabs para indentación
- [ ] `status::` usa valores en inglés sin `#` prefix
- [ ] Bloques importantes incluyen `project:: [[CALAIRE-EA]]`
- [ ] Links a MOCs usan formato `[[Nombre de Página]]`
