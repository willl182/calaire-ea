# Problem Record: Revisión de Informe de Resultados EA

**Severity**: Alta

## El Probleble

Durante la revisión de fases con el subagent `revisor-fase`, se identificaron problemas de consistencia y formato en los documentos generados que pueden afectar la implementación correcta de los cambios en la plantilla Rmd:

1. **Citaciones incorrectas de cláusulas ISO:** En la matriz de requisitos (fase 1), ISO 13528 §9.2.1 está citado incorrectamente; debe ser §9.4.1. Esto afecta la trazabilidad normativa.

2. **Severidad no normalizada:** Se usan "Alta/Media/Baja" y también "Bloqueantes" sin un esquema claro de mapeo, lo que dificulta la priorización de acciones.

3. **Errores de formato Markdown/Rmd:**
   - Bloques de código anidados con triples backticks (rompen el render)
   - Cierres de fences duplicados
   - Símbolos "|" no escapados en tablas (rompen la estructura)
   - Delimitadores de LaTeX incompletos en fórmulas matemáticas

4. **Inconsistencia entre texto y código:** El criterio de estabilidad en el texto usa `c_estab` pero el código Rmd fija `0.3 * sigma_pt` (criterio típico de homogeneidad). Esto puede inducir a aplicar el criterio incorrecto.

5. **Acciones no accionables:** Placeholder como `[ESPECIFICAR OPCIÓN]` sin criterios claros de responsabilidad ni verificación.

6. **Dependencias no documentadas:** Los ejemplos de código Rmd usan funciones como `robust_mean`, `robust_sd` sin especificar dependencias de paquetes o preparación del entorno.

## Correcciones Solicitadas por Usuario

- "quiero que me des un documento md donde van todos los cambios explicados y el ejemplo de lo que deberia de ir" - Se creó documento completo de instrucciones para Rmd con ejemplos

## La Solución / Workaround

1. Crear checklist de correcciones pendientes integrado en el documento principal de instrucciones
2. Documentar hallazgos de revisión de fases con severidad y estado pendiente
3. Proporcionar esquema claro de priorización (Prioridad 1-7 con cronograma sugerido)
4. Añadir sección de dependencias y objetos requeridos para ejemplos Rmd

## Learninges a Preservar

1. **Validar siempre el formato:** Antes de generar documentos con bloques de código, verificar que usen fences de 4 backticks, no triples.

2. **Mantener consistencia de notación:** Alinear fórmulas matemáticas en texto, código Rmd y LaTeX para evitar contradicciones.

3. **Normalizar severidades:** Usar esquema consistente (Alta/Media/Baja) en lugar de mezclar con "Bloqueantes" o descripciones verbales.

4. **Verificar referencias normativas:** Cross-check las cláusulas citadas contra la norma vigente para asegurar exactitud.

5. **Documentar dependencias:** Siempre especificar paquetes requeridos y objetos mínimos para que el código sea ejecutable.

6. **Escapar caracteres especiales:** En tablas Markdown, escapar caracteres como "|" que rompen la estructura.

7. **Evitar placeholders sin criterios:** Si no se puede especificar una acción concreta, dejar el placeholder pero documentar quién es responsable de definirlo y cuándo.
