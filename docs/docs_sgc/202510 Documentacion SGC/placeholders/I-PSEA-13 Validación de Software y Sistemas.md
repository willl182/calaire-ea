## Descripción

Instructivo para la validación formal del aplicativo R/Shiny utilizado en el esquema de EA. Define el expediente de validación: especificación de requisitos, plan de pruebas, datasets de referencia, comparación de resultados esperados vs. obtenidos, tolerancia de aceptación (1e-9 para cálculos numéricos), validación cruzada R vs. Python, documentación de bugs y correcciones, control de versiones del software, y revalidación ante cambios. Brecha crítica identificada en la evaluación del SGC.
