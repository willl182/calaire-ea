Arquitectura del Flujo

Dashboard de rondas y formulario reactivo para participantes

Este prototipo separa dos funciones: la configuración operativa de rondas por parte del organizador y el diligenciamiento del reporte por parte del participante. La ronda define contaminantes, número de niveles y unidad sugerida. El formulario del participante se adapta automáticamente a esa configuración.

Notas de uso

  • La ronda guardada define la estructura del formulario del participante.
  • Sin backend, el almacenamiento es local al navegador del organizador.
  • La exportación de rondas en JSON permite mover la configuración a otra instancia publicada.
  • Cuando haya backend, esta misma estructura puede persistirse en base de datos sin cambiar el modelo conceptual.
